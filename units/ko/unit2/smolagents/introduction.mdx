## 모듈 개요

이 모듈에서는 `smolagents`를 사용한 지능형 에이전트 구축의 핵심 개념과 실용적 전략을 포괄적으로 다룹니다.

많은 오픈소스 프레임워크가 있는 만큼, `smolagents`의 구성 요소와 기능을 이해하고 다른 솔루션이 더 적합할 수 있는 경우를 판단하는 것이 중요합니다.

소프트웨어 개발 작업을 위한 코드 에이전트, 모듈식 함수 기반 워크플로우를 위한 도구 호출 에이전트, 정보를 접근하고 종합하는 검색 에이전트 등 주요 에이전트 유형을 살펴볼 것입니다.

또한 다중 에이전트의 조율과 비전 기능 및 웹 브라우징 통합을 다루며, 이를 통해 동적이고 상황 인식이 가능한 애플리케이션의 새로운 가능성을 열어볼 것입니다.

이 유닛에서는 Unit 1의 에이전트 알프레드가 다시 등장합니다. 이번에는 `smolagents` 프레임워크를 내부 동작에 사용합니다. 알프레드가 다양한 작업을 수행하면서 이 프레임워크의 핵심 개념을 함께 살펴볼 것입니다. 웨인 가족 🦇이 자리를 비운 사이 웨인 저택에서 파티를 준비하느라 알프레드는 할 일이 많습니다. `smolagents`로 이러한 작업을 처리하는 알프레드의 여정을 함께 살펴보세요!

<Tip>

이 유닛에서는 `smolagents` 라이브러리로 AI 에이전트를 구축하는 방법을 배웁니다. 여러분의 에이전트는 데이터를 검색하고, 코드를 실행하고, 웹 페이지와 상호작용할 수 있게 됩니다. 또한 여러 에이전트를 결합해 더 강력한 시스템을 만드는 방법도 배우게 됩니다.

</Tip>

## 목차

이 `smolagents` 유닛에서는 다음 내용을 다룹니다:

### 1️⃣ [smolagents를 사용하는 이유](./why_use_smolagents)

`smolagents`는 애플리케이션 개발에 사용할 수 있는 많은 오픈소스 에이전트 프레임워크 중 하나입니다. `LlamaIndex`와 `LangGraph` 같은 대안도 있으며, 이 과정의 다른 모듈에서 다룹니다. `smolagents`는 특정 사용 사례에 적합한 여러 핵심 기능을 제공하지만, 프레임워크를 선택할 때는 항상 모든 옵션을 고려해야 합니다. 프로젝트 요구사항에 따라 현명한 결정을 내릴 수 있도록 `smolagents` 사용의 장단점을 살펴볼 것입니다.

### 2️⃣ [CodeAgents](./code_agents)

`CodeAgents`는 `smolagents`의 주요 에이전트 유형입니다. JSON이나 텍스트 대신 Python 코드를 생성해 작업을 수행합니다. 이 모듈에서는 그 목적, 기능, 작동 방식을 살펴보고 실습 예제로 기능을 보여줍니다.

### 3️⃣ [ToolCallingAgents](./tool_calling_agents)

`ToolCallingAgents`는 `smolagents`가 지원하는 두 번째 에이전트 유형입니다. Python 코드를 생성하는 `CodeAgents`와 달리, 이 에이전트는 시스템이 파싱하고 해석해야 하는 JSON/텍스트 블롭에 의존합니다. 이 모듈에서는 기능, `CodeAgents`와의 주요 차이점을 다루고 예제로 사용법을 설명합니다.

### 4️⃣ [Tools](./tools)

Unit 1에서 보았듯이, 도구는 LLM이 에이전트 시스템 내에서 사용할 수 있는 함수이며 에이전트 동작의 필수 구성 요소입니다. 이 모듈에서는 도구 생성 방법, 구조, `Tool` 클래스나 `@tool` 데코레이터를 사용한 다양한 구현 방법을 다룹니다. 또한 기본 도구 상자, 커뮤니티와 도구 공유 방법, 커뮤니티가 기여한 도구를 에이전트에서 사용하기 위해 로드하는 방법도 배웁니다.

### 5️⃣ [Retrieval Agents](./retrieval_agents)

검색 에이전트는 모델이 지식 베이스에 접근할 수 있게 해, 여러 소스에서 정보를 검색하고 종합하고 검색할 수 있게 합니다. 효율적인 검색을 위해 벡터 저장소를 활용하고 **검색 증강 생성(RAG)** 패턴을 구현합니다. 이러한 에이전트는 메모리 시스템을 통해 대화 맥락을 유지하면서 웹 검색과 사용자 지정 지식 베이스를 통합하는 데 특히 유용합니다. 이 모듈에서는 강력한 정보 검색을 위한 대체 메커니즘을 포함한 구현 전략을 살펴봅니다.

### 6️⃣ [Multi-Agent Systems](./multi_agent_systems)

강력한 다중 에이전트 시스템을 구축하려면 여러 에이전트를 효과적으로 조율하는 것이 중요합니다. 웹 검색 에이전트와 코드 실행 에이전트처럼 서로 다른 기능을 가진 에이전트를 결합하면 더 정교한 솔루션을 만들 수 있습니다. 이 모듈에서는 효율성과 신뢰성을 극대화하기 위한 다중 에이전트 시스템의 설계, 구현, 관리에 중점을 둡니다.

### 7️⃣ [Vision and Browser agents](./vision_agents)

비전 에이전트는 **비전-언어 모델(VLM)**을 통합해 시각적 정보를 처리하고 해석할 수 있도록 기존 에이전트 기능을 확장합니다. 이 모듈에서는 VLM 기반 에이전트를 설계하고 통합하는 방법을 살펴보며, 이미지 기반 추론, 시각적 데이터 분석, 멀티모달 상호작용과 같은 고급 기능을 활용합니다. 또한 비전 에이전트를 사용해 웹을 탐색하고 정보를 추출할 수 있는 브라우저 에이전트를 구축할 것입니다.
