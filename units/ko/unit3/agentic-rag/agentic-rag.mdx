# Agentic Retrieval Augmented Generation (RAG)

이번 유닛에서는 Agentic RAG를 활용해 Alfred가 멋진 갈라 파티를 준비할 수 있도록 돕는 방법을 살펴보겠습니다.

<Tip>이전 유닛에서 Retrieval Augmented Generation(RAG)과 agentic RAG에 대해 이미 다루었으니, 개념이 익숙하다면 이 부분을 건너뛰셔도 좋습니다.</Tip>

LLM은 방대한 데이터로 학습되어 일반적인 지식을 습득합니다.
하지만 LLM의 세계 지식은 항상 최신이거나 관련성이 높지 않을 수 있습니다.
**RAG는 여러분의 데이터에서 관련 정보를 찾아 LLM에 전달함으로써 이 문제를 해결합니다.**

![RAG](https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit2/llama-index/rag.png)

이제 Alfred가 어떻게 일하는지 생각해봅시다:

1. 우리는 Alfred에게 갈라 파티 준비를 부탁했습니다
2. Alfred는 최신 뉴스와 날씨 정보를 찾아야 합니다
3. Alfred는 손님 정보를 구조화하고 검색해야 합니다

Alfred가 집안의 정보를 찾아야 도움이 될 수 있듯, 어떤 에이전트든 관련 데이터를 찾고 이해할 수 있는 방법이 필요합니다.
**Agentic RAG는 에이전트가 여러분의 데이터에 대한 질문에 답할 수 있게 해주는 강력한 방법입니다.** 우리는 Alfred에게 다양한 도구를 제공해 질문에 답하도록 도울 수 있습니다.
하지만 단순히 문서 위에서 자동으로 답변하는 것이 아니라, Alfred는 질문에 답하기 위해 다른 도구나 플로우를 자유롭게 선택할 수 있습니다.

![Agentic RAG](https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit2/llama-index/agentic-rag.png)

이제 **agentic RAG 워크플로우를 직접 만들어봅시다!**

먼저, 초대 손님에 대한 최신 정보를 검색하는 RAG 도구를 만들고, 이어서 웹 검색, 날씨 업데이트, Hugging Face Hub 모델 다운로드 통계 도구를 개발합니다. 마지막으로 모든 것을 통합해 agentic RAG 에이전트를 완성할 것입니다!
