# 관찰: 피드백 통합과 적응

관찰(Observation)은 **에이전트가 자신의 행동 결과를 인식하는 방법**입니다.

관찰은 에이전트의 사고 과정에 중요한 정보를 제공하며, 다음 행동을 안내합니다.

관찰은 **환경에서 오는 신호**입니다. 예: API 데이터, 에러 메시지, 시스템 로그 등. 이 신호가 다음 사이클의 사고를 이끕니다.

관찰 단계에서 에이전트는:
- **피드백 수집:** 행동이 성공했는지(또는 실패했는지) 데이터/확인 수신
- **결과 추가:** 새로운 정보를 기존 맥락에 통합(메모리 업데이트)
- **전략 적응:** 갱신된 맥락을 바탕으로 다음 사고/행동을 조정

예를 들어, 날씨 API가 *"부분적으로 흐림, 15°C, 습도 60%"*를 반환하면, 이 관찰 결과가 에이전트의 메모리(프롬프트 끝)에 추가됩니다.

에이전트는 이 정보를 바탕으로 추가 정보가 필요한지, 최종 답을 낼 준비가 됐는지 결정합니다.

이런 **반복적 피드백 통합 덕분에 에이전트는 목표에 맞게 동적으로 적응**하며, 실제 결과에 따라 계속 학습/조정합니다.

관찰은 **다양한 형태**를 가질 수 있습니다. 예: 웹페이지 텍스트, 로봇 팔 위치, 도구 실행 로그 등.

| 관찰 유형 | 예시 |
|-----------|------|
| 시스템 피드백 | 에러 메시지, 성공 알림, 상태 코드 등 |
| 데이터 변화 | DB 업데이트, 파일 시스템 변경, 상태 변화 등 |
| 환경 데이터 | 센서 값, 시스템 메트릭, 리소스 사용량 등 |
| 응답 분석 | API 응답, 쿼리 결과, 계산 결과 등 |
| 시간 기반 이벤트 | 마감 도달, 예약 작업 완료 등 |

## 결과는 어떻게 추가되는가?

행동 후 프레임워크는 다음 순서로 진행:
1. **행동 파싱:** 호출할 함수/인자 식별
2. **행동 실행**
3. **결과를 관찰로 추가**
